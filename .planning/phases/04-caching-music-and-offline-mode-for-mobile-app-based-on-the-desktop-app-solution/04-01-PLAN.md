# Locked decisions from CONTEXT.md:
- Store audio in Documents directory (user-accessible, persistent)
- Flat file organization with track IDs: audio/{trackId}.{ext}
- Keep original file extension (mp3, m4a, flac, etc.)
- Default max cache size: 2GB
- LRU eviction when cache is full
- Track cache metadata in MobileDatabase (existing tracks table extended)

<task type="auto">
  <name>Task 2: Create MobileCacheService</name>
  <files>mobile/services/MobileCacheService.ts</files>
  <action>
Create MobileCacheService with:

1. Constants:
   - AUDIO_DIR = `${FileSystem.documentDirectory}audio/`
   - DEFAULT_MAX_SIZE = 2 * 1024 * 1024 * 1024 (2GB)

2. Properties:
   - maxSizeBytes: number (configurable)
   - activeDownloads: Map<string, AbortController>

3. Methods:
   - downloadTrack(track: Track): Promise<void>
     - Ensure audio directory exists
     - Extract extension from streamUrl (default .mp3)
     - Use FileSystem.createDownloadResumable for download with progress
     - Save entry to audio_cache table
     - Emit 'download-progress' event

   - downloadAlbum(tracks: Track[]): Promise<void>
     - Download all tracks in parallel (Promise.all)
     - Track overall progress

   - deleteTrack(trackId: string): Promise<void>
     - Delete file from filesystem
     - Remove from audio_cache table

   - deleteAlbum(albumId: string): Promise<void>
     - Get all tracks for album
     - Delete each track

   - clearCache(): Promise<void>
     - Delete all files in audio directory
     - Clear audio_cache table

   - getCacheStatus(trackId: string): Promise<boolean>
     - Check if track is cached

   - getCacheSize(): Promise<number>
     - Get total cache size from database

   - ensureCacheSpace(requiredBytes: number): Promise<void>
     - LRU eviction: getOldestCacheEntries, delete until space available

4. Helper methods:
   - getExtension(url: string): string
   - getFilePath(trackId: string, ext: string): string
   - async ensureDirectory(): Promise<void>

Import FileSystem from 'expo-file-system'. Use EventEmitter for progress events.
