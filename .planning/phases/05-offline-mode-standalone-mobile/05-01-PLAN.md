Create CacheFab.tsx — a floating action button in the bottom-right corner for bulk download operations in standalone mode.

Component spec:
- Position: bottom-right, 16px from edges, above the player bar (use bottom: playerBarHeight + 16, default ~76px)
- Size: 56px diameter (standard FAB)
- Visibility: Only in standalone mode, hidden in remote and offline modes
- Icon states:
  - Not all cached: Download icon (from lucide-react-native Download)
  - All cached: Check icon (from lucide-react-native Check)
  - Downloads in progress: Animated spinner (from lucide-react-native Activity)
  - Show progress bar below FAB when downloadingTrackIds.size > 0

Behavior:
- On press: Show ActionSheet with three options:
  1. "Download All Cached" — downloads all uncached tracks in the collection (only show if not all cached)
  2. "Download All Visible" — downloads all visible items on current screen (always show)
  3. "Cancel Downloads" — cancels all active downloads in downloadingTrackIds (only show if downloads in progress)
- The FAB passes onDownloadAllCached, onDownloadAllVisible, onCancelDownloads, and onClose callbacks to the ActionSheet

Props interface:
```typescript
interface CacheFabProps {
    visible: boolean  // show only in standalone mode
    isAllCached: boolean
    downloadProgress: number | null  // 0-1 for indeterminate, null when not downloading
    onDownloadAllCached: () => void
    onDownloadAllVisible: () => void
    onCancelDownloads: () => void
}
```

Import: TouchableOpacity, View, StyleSheet, Text from react-native; Download, Check, Activity from lucide-react-native; useTheme, useStore from appropriate paths; ActionSheet from './ActionSheet'

Do NOT implement the download logic here — just call the callbacks. The download logic lives in the screens that use this component (Plan 05-02).
